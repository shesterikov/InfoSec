# Практическая работа 8

## SQL инъекции и способы защиты от них

### Цель работы

Познакомиться с SQL инъекциями и способами защиты от них

### Технические требования:

- Наличие интернет-соединения
- Наличие [Docker](https://docs.docker.com/desktop/) и [Docker Compose](https://docs.docker.com/compose/install/)

### Ход работы:

1. Запустите приложение при помощи команды
   `docker compose up -d --build`

2. Убедитесь, что приложение запущено корректно.
   В случае успешного запуска при переходе на [страницу](http://localhost:8080/) отобразится следующая страница
   ![Рисунок 8.1 - Страница входа](/assets/task-08/1.png)

3. Выполните вход, используя следующие данные

```
username: user1
password: 22221
```

Убедитесь, что вход не выполнен, а на странице отображается следующее сообщение
![Рисунок 8.2 - Вход не выполнен](/assets/task-08/2.png)

4. Выполните вход, используя корректные данные

```
username: user1
password: password1
```

Убедитесь, что входе выполнен, а на странице отображается следующее сообщение
![Рисунок 8.3 - Вход выполнен успешно](/assets/task-08/3.png)

5. Выполните вход, используя в качестве `username` следующее значение

```
' OR '1'='1' --
```

Поле `password` при этом оставьте пустым

![Рисунок 8.4 - Вход с использование SQLi](/assets/task-08/4.png)
В случае успеха произойдет обход аутентификации (bypass authentication)
![Рисунок 8.5 - Обход аутентификации](/assets/task-08/5.png)

6. Выполните проверку серверных логов при помощи команды `docker logs app`
   ![Рисунок 8.6 - Серверные логи](/assets/task-08/6.png)
   Ознакомьтесь с тем, в каком виде при запросах к серверу были представлены изначальные и модифицированный запросы

7. Выполните SQLi для модификации пароля пользователя `admin`

8. Модифицируйте исходный код `main.go` для защиты от SQLi. С этой целью воспользуйтесь параметризированными запросами и примерами, представленными в документации
