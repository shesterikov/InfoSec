# Практическая работа 1

## Знакомство с протоколами HTTP и HTTPS

### Цель работы

Ознакомиться с различиями в работе протоколов HTTP и HTTPS при помощи анализатора пакетов Wireshark

### Технические требования:

- Наличие интернет-соединения
- Наличие [Wireshark](https://www.wireshark.org/)

### Ход работы:

1. Запустите Wireshark
   ![Рисунок 1.1 - Главный экран Wireshark](/assets/task-01/1.png)

2. Запустите перехват сетевых пакетов в Wireshark
   ![Рисунок 1.2 - Перехват сетевых пакетов при помощи Wireshark
](/assets/task-01/2.png)

3. В интернет браузере перейдите на любой сайт, не использующий протокол HTTP, к примеру сайт правительства [РФ](http://government.ru/)

4. В Wireshark выполните перехват сетевых пакетов для заданного узла, при необходимости воспользовавшись утилитой `nslookup` для определения IP адреса целевого узла
   ![Рисунок 1.3 - Отфильтрованные сетевые пакеты сайта Правительства РФ](/assets/task-01/3.png)

5. Выберите любой пакет `GET / HTTP/1.1`
   ![Рисунок 1.4 - Выбранный HTTP пакет](/assets/task-01/4.png)

6. Откройте HTTP Stream для просмотра
   ![Рисунок 1.5 - Меню выбора для слежения за HTTP Stream](/assets/task-01/5.png)

7. Убедитесь, что заданный ресурс не использует шифрование, а содержимое пакетов представлено в открытом виде
   ![Рисунок 1.6 - HTTP Stream с незашифрованным контентом](/assets/task-01/6.png)

8. В интернет браузере перейдите на любой сайт, использующий протокол HTTPS, к примеру сайт Расписание [НПИ](https://schedule.npi-tu.ru/)

9. В Wireshark выполните перехват сетевых пакетов для заданного узла, при необходимости воспользовавшись утилитой `nslookup` для определения IP адреса. Убедитесь, что пакеты, передаваемые между клиентом и сервером зашифрованы
   ![Рисунок 1.7 - HTTP Stream с зашифрованным контентом
](/assets/task-01/7.png)

10. Выполните анализ TLS рукопожатия (TLS Handshake). Отразите в отчете фазы Client Hello, Server Hello и то, какими данными клиент и сервер обмениваются в это время. Если имеет место быть повышение версии TLS - опишите, как это происходит.

### Контрольные вопросы:

1. Что такое HTTP / HTTPS?
2. Какой порт по умолчанию используется протоколом HTTP?
3. Какой порт по умолчанию используется протоколом HTTPS?
4. Что такое TLS?
5. Что такое TLS Handshake? Какие фазы для него характерны?
6. Каким образом обеспечивается организация шифрованного соединения при использовании HTTPS протокола?
